#### 3.1.3 hooks.py：中间层激活提取

```python
class ActivationExtractor:
    """通过 forward hook 提取指定层的激活输出。"""

    def __init__(self, model: nn.Module, layer_names: list[str] | None = None):
        """layer_names=None 时自动发现所有 Conv2d/Linear 层。"""

    def extract(self, x: Tensor) -> dict[str, Tensor]:
        """前向传播一次，返回 {层名: 激活张量}。
        Conv2d 输出做全局平均池化，压缩为 (batch, channels)。
        Linear 输出保持 (batch, features)。
        """
```

算法参考 NeuraL-Coverage 的 tool.py 的 hook 模式，但使用 model.named_modules() 获取完整路径名（如 "layer4.1.conv2"）而非 NeuraL-Coverage 的计数命名（如 "Conv2d-15"），路径名更稳定且可读。

### 3.2 differential — 差分测试（研究内容 1）

#### 3.2.1 ensemble.py：多模型管理

```python
class ModelEnsemble:
    """管理多个预训练模型，提供统一的推理接口。"""

    def __init__(self, model_names: list[str], dataset: str, device: torch.device):
        """加载所有模型，例如 ["resnet50", "vgg16_bn", "mobilenet_v2"]。"""

    def predict(self, x: Tensor) -> list[ModelPrediction]:
        """对同一输入运行所有模型，返回各模型的预测结果。"""

    def consensus_label(self, x: Tensor) -> int | None:
        """返回所有模型一致同意的类别标签，若无共识返回 None。"""

    def find_disagreement(self, predictions: list[ModelPrediction]) -> bool:
        """判断目标模型 F_j 是否与其他模型产生分歧。"""
```

#### 3.2.2 objective.py：差分目标函数

```python
class DifferentialObjective:
    """计算 obj_1(x) = Σ_{k≠j} F_k(x)[c] - λ_1·F_j(x)[c]"""

    def __init__(self, ensemble: ModelEnsemble, target_idx: int, lambda1: float = 1.0): ...

    def __call__(self, x: Tensor, consensus_label: int) -> Tensor:
        """返回标量目标值（越大越好 → 目标模型越偏离共识）。x 需要 requires_grad=True。"""

    def gradient(self, x: Tensor, consensus_label: int) -> Tensor:
        """返回 ∇_x obj_1，与 x 同 shape。"""
```

这是框架与三个开源项目的关键区别：三个开源项目都用图像变换（旋转、亮度等）做变异，本框架直接用 obj_1 的梯度驱动变异方向。

### 3.3 neurons — 关键神经元（研究内容 2）

#### 3.3.1 profiler.py：关键神经元识别

```python
@dataclass
class NeuronProfile:
    layer_name: str
    neuron_idx: int
    cl: float                    # 关键度水平
    cl_class: dict[int, float]   # 各类别的关键度

@dataclass
class CriticalNeuronSet:
    neurons: list[NeuronProfile]           # 所有关键神经元
    by_layer: dict[str, list[int]]         # 按层索引
    by_class: dict[int, list[NeuronProfile]]  # 按类别索引
    threshold: float                        # 使用的阈值 ε

class NeuronProfiler:
    """离线分析模型，识别关键神经元集合 D_en。"""

    def __init__(self, model: nn.Module, extractor: ActivationExtractor,
                 activation_threshold: float = 0.1,
                 critical_threshold: float = 0.75): ...

    def profile(self, train_loader: DataLoader) -> CriticalNeuronSet:
        """遍历训练集，计算每个神经元的 cl(n,T)。
        算法：
        1. 对每个 batch，提取各层激活
        2. 激活值做 min-max 归一化到 [0,1]（按层、按神经元维度）
        3. 对每个神经元，统计激活值 > activation_threshold 的样本数
        4. cl = 激活样本数 / 总样本数
        5. 筛选 cl > critical_threshold 的神经元进入 D_en
        同理对每个类别子集 T_c 计算 cl_class。
        """
```

算法逻辑参考 CriticalFuzz 的 get_critial_neuron.py，但有两点不同：
- 使用 PyTorch hook 而非 Keras 的 layer output
- 使用模型的完整层路径名而非计数命名

#### 3.3.2 coverage.py：覆盖准则

```python
class CNCovTracker:
    """跟踪关键神经元覆盖率。"""

    def __init__(self, critical_set: CriticalNeuronSet): ...

    def update(self, activations: dict[str, Tensor]) -> list[str]:
        """输入一组激活，更新覆盖状态。返回本次新覆盖的神经元标识列表。"""

    @property
    def cncov(self) -> float:
        """当前 CNCov = 已覆盖关键神经元数 / |D_en|"""

    @property
    def cccov(self) -> dict[int, float]:
        """各类别的 CCCov。"""

    @property
    def uncovered(self) -> list[NeuronProfile]:
        """尚未覆盖的关键神经元列表。"""
```

```python
class CoverageObjective:

        """返回标量目标值。最大化此值 → 激活未覆盖的关键神经元。"""
```

#### 3.4.2 feature.py：轻量路线

```python
class FeatureDistance:
    """在预训练模型的中间层特征空间中计算余弦距离。
    取模型某一层（如 ResNet 的 avgpool 前一层）的输出作为特征表示，

```

不依赖外部模型，直接复用测试目标模型自身的中间层表示。计算开销极低（一次额外前向传播），适合作为默认实现。

#### 3.4.3 clip.py：重管线

```python
class CLIPDistance:
    """使用 CLIP 视觉编码器计算图像级语义相似度。
    将原始/变异样本分别过 CLIP vision encoder，计算 embedding 余弦距离。
    """

    def __init__(self, model_name: str = "ViT-B-32", device: torch.device): ...
```

与 NSGen 的方式不同：NSGen 先做神经元归因 → MILAN 文本描述 → CLIP 文本编码器比较，管线很重。这里直接用 CLIP 的视觉编码器比较两张图的 embedding，省去了 MILAN 和文本编码的开销，同时保留了 CLIP 预训练语义空间的优势。

### 3.5 optimize — 联合优化（研究内容 4）

#### 3.5.1 joint.py：联合目标函数

```python
class JointObjective:
    """obj_total(x) = obj_1(x) + λ_2·obj_cov(x) - λ_3·obj_sem(x)"""

    def __init__(self, diff_obj: DifferentialObjective,
                 cov_obj: CoverageObjective,
                 sem_obj: SemanticObjective,
                 lambda2: float = 0.5,
                 lambda3: float = 0.3): ...

    def __call__(self, x: Tensor, consensus_label: int) -> Tensor:
        """返回联合目标值。"""

    def gradient(self, x: Tensor, consensus_label: int) -> Tensor:
        """返回 ∇_x obj_total。通过 autograd 自动计算，无需手动推导。"""
```

#### 3.5.2 operator.py：T(·) 约束算子

```python
class ConstraintOperator:
    """对原始梯度施加约束，确保变异方向合理。
    T(∇) = semantic_project(domain_constrain(∇))
    """

    def __init__(self, epsilon: float = 0.03,
                 domain_constraints: list[str] | None = None): ...

    def __call__(self, grad: Tensor, x: Tensor) -> Tensor:
        """
        1. 域约束：将梯度裁剪到 L∞ 球内（epsilon），保证物理可实现性
        2. 像素裁剪：确保 x + grad 仍在 [0,1] 范围内
        后续可扩展：特征空间投影（待研究方案待办第一步确定后实现）
        """
```

初始实现使用 PGD 式的投影约束（L∞ 裁剪 + 像素范围裁剪）。这已经能保证基本的物理可实现性。语义投影部分待研究方案待办第一步确定 T(·) 技术路线后再细化。

#### 3.5.3 feedback.py：动态反馈

```python
@dataclass
class FeedbackState:
    delta_cncov: float           # 本轮覆盖增长
    rft: float                   # 本轮缺陷触发率
    d_sem_mean: float            # 本轮平均语义偏移

class DynamicFeedback:
    """根据测试状态自适应调整 λ_2 和 λ_3。"""

    def __init__(self, delta_cov_threshold: float = 0.01,
                 delta_sem_threshold: float = 0.5,
                 delta_fault_threshold: float = 0.05,
                 window_size: int = 5): ...

    def update(self, state: FeedbackState) -> tuple[float, float]:
        """输入当前轮次的反馈指标，返回调整后的 (λ_2, λ_3)。
        规则：
        - ΔCNCov 连续 window_size 轮 < δ_cov → 降低 λ_2
        - d̄_sem > δ_sem → 提升 λ_3
        - RFT < δ_fault → 提升 λ_2
        """
```

#### 3.6.2 runner.py：主循环

```python
@dataclass
class FuzzConfig:
    max_iterations: int = 10000
    step_size: float = 0.01      # 梯度步长 s
    batch_size: int = 32
    seed_pool_size: int = 1000
    target_cncov: float = 0.9    # CNCov 目标值
    stagnation_window: int = 50  # 连续无进展轮数上限

class FuzzRunner:
    """主测试循环，串联所有模块。"""

    def __init__(self, config: FuzzConfig,
                 joint_obj: JointObjective,
                 operator: ConstraintOperator,
                 coverage: CNCovTracker,
                 feedback: DynamicFeedback,
                 ensemble: ModelEnsemble,
                 seed_pool: SeedPool,
                 semantic_fn: SemanticDistance): ...

    def run(self) -> FuzzReport:
        """
        主循环：
        for iteration in range(max_iterations):
            1. 从种子池选取一批种子
            2. 对每个种子：
               a. 计算 ∇_x obj_total（需要 requires_grad）
               b. 通过 T(·) 约束算子修正梯度
               c. 变异：x' = x + s · T(∇)
               d. 多模型推理，检测分歧 → 记录缺陷
               e. 提取激活，更新覆盖状态
               f. 若覆盖了新关键神经元 → 加入种子池
               g. 计算语义距离
            3. 汇总本轮 ΔCNCov、RFT、d̄_sem
            4. 动态调整 λ_2、λ_3
            5. 检查终止条件
        返回 FuzzReport
        """
```

支持分阶段运行：当某些模块尚未实现时，可以传入 None 或 dummy 实现。例如第一阶段只有差分目标，coverage 和 semantic 传入空实现。

## 四、分阶段实现计划

### Phase 1：基座 + 差分测试

实现：core 全部 + differential 全部 + engine（简化版，仅差分目标）

此阶段的 runner 只计算 obj_1 梯度做变异，不涉及覆盖和语义。相当于一个梯度驱动的 DeepXplore 复现。

产出的实验：
- **梯度驱动差分 fuzzing vs 随机 fuzzing**：缺陷发现数量、效率曲线
- **多模型基线一致率**：三个模型在 CIFAR-10 上的预测一致性统计
- **差分信号有效性**：obj_1 值与实际分歧的相关性

### Phase 2：+ 关键神经元

实现：neurons 全部，engine 扩展为 obj_1 + obj_cov

产出的实验：
- **关键神经元分布分析**：cl(n,T) 直方图、阈值敏感性曲线
- **CNCov vs 传统 NC**：对同一测试集的覆盖率对比
- **差分+覆盖 vs 纯差分 vs 随机**：三组对比，看覆盖引导是否提升缺陷发现效率

### Phase 3：+ 语义约束

实现：semantic（先实现轻量路线 feature.py），engine 扩展为完整 obj_total

产出的实验：
- **有/无语义约束的对比**：生成样本的语义偏移分布、视觉质量
- **完整框架 vs 无语义 vs 纯差分 vs 随机**：四组消融
- **语义距离阈值敏感性分析**

### Phase 4：+ 动态反馈

实现：optimize/feedback.py，engine 接入动态 λ 调整

产出的实验：
- **动态反馈 vs 静态权重**：覆盖率增长效率、缺陷发现数量对比
- **λ 变化曲线分析**：展示框架在不同阶段自动调整策略的行为
- **与 baseline 方法的完整对比**（CriticalFuzz、NSGen 的结果作为参照）

### Phase 5：评估与优化

实现：evaluate 全部，补充 CLIP 语义路线（若决定采用）

产出的实验：
- **完整评估报告**：缺陷发现能力、覆盖效率、样本质量、方法稳定性
- **跨模型泛化性**：在不同目标模型上的表现
- **跨数据集泛化性**：CIFAR-10 → ImageNet

## 五、实验设计总览

### 实验与研究内容的映射

| 实验 | 研究内容 | 自变量 | 因变量 | baseline |
|------|----------|--------|--------|----------|
| 差分信号有效性 | RC1 | 迭代次数、λ_1 | 分歧触发率、缺陷数 | 随机 fuzzing |
| 关键神经元分布 | RC2 | 阈值 ε、模型 | D_en 规模、层分布 | — |
| CNCov vs NC | RC2 | 测试集 | 覆盖率、区分度 | 传统 NC、KMNC |
| 语义约束效果 | RC3 | 有/无约束、阈值 | 语义偏移、样本质量 | 无约束 |
| 联合优化消融 | RC4 | 目标组合 | 缺陷数、覆盖率、语义距离 | 各单目标/双目标 |
| 动态反馈效果 | RC4 | 动态/静态 | 效率曲线 | 固定 λ |
| 方法综合对比 | RC5 | 方法 | 全指标 | CriticalFuzz, random |

### 核心评估指标

1. 缺陷发现数量（#defects）：目标模型预测与参考模型不一致的样本数
2. CNCov / CCCov：关键神经元覆盖率
3. 缺陷发现效率：达到 N 个缺陷所需迭代次数
4. 覆盖效率：达到目标 CNCov 所需迭代次数
5. 平均语义偏移（d̄_sem）：生成样本与原始种子的平均语义距离
6. 缺陷触发率（RFT）：每轮生成样本中缺陷占比

## 六、验证方式

1. **单元测试**：每个模块的核心函数编写 pytest 测试（梯度计算的数值验证、覆盖率计算的正确性等）
2. **冒烟测试**：用 CIFAR-10 的小子集（100 张）跑完整流程，确认管线不报错
3. **基线对齐**：Phase 1 的差分 fuzzing 结果应与 DeepXplore 论文的趋势一致；Phase 2 的关键神经元分布应与 CriticalFuzz 的结果定性一致
4. **消融验证**：每增加一个模块，指标应有可观测的变化（不一定都是提升，但变化方向应与预期一致）
