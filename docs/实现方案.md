# mfuzz 实现方案

## 一、项目概况

本项目实现研究方案中提出的"多维反馈驱动的中间层安全分析与测试框架"。框架名为 mfuzz，融合差分测试、关键神经元覆盖、语义约束三个维度，通过梯度驱动的联合优化自动生成能触发 DNN 预测缺陷的测试样本。

三个开源项目（CriticalFuzz、NeuraL-Coverage、NSGen）作为算法参考。实现要求：现代工具链（Python 3.13、PyTorch 2.x、uv），全部重新设计和编写，不照搬原代码。

## 四、当前实现状态

### 已完成

Phase 1（差分测试基座）和 Phase 2（关键神经元覆盖引导）已完成编码和冒烟测试。

具体包括：core 全部模块（模型加载、激活提取、数据集加载）、differential 全部模块（多模型管理、差分目标函数 obj_1 及梯度）、neurons 全部模块（离线 profiling、CNCov 跟踪、覆盖目标函数 obj_cov）、engine 全部模块（种子池、主循环）、evaluate/report.py（缺陷图片保存、指标 JSON、曲线图）、PGD 式约束算子。

入口脚本 run_fuzz.py 通过 TOML 配置控制运行模式：neurons.enabled=false 为纯差分模式，neurons.enabled=true 为差分+覆盖模式。已加入全局随机种子保证实验可复现，profiling 结果支持缓存。

### 已发现的问题

实验中 CNCov 从第 1 轮即为 1.0（critical_threshold=0.75 时关键神经元占 97%，一次前向传播就全部覆盖）。需要调高阈值参数使关键集合具有区分度。用户已将 critical_threshold 调为 0.25，待重新跑实验验证。

diff-only（502 缺陷）和 diff+cov（482 缺陷）在旧阈值下结果几乎无差异，原因是覆盖梯度始终为零——覆盖引导实际未生效。

## 五、模块设计要点

本节记录各模块的关键设计决策，作为实现参考。已完成的模块记录实际实现的要点，待实现的模块记录设计方向。

### 5.1 core

**models.py**：支持 ResNet50、VGG16-BN、MobileNetV2。ImageNet 模型使用 torchvision 预训练权重（ResNet50/MobileNetV2 用 IMAGENET1K_V2，VGG16-BN 用 IMAGENET1K_V1）。CIFAR-10 模型需指定权重路径。

**hooks.py**：ActivationExtractor 通过 forward hook 提取中间层激活。自动发现所有 Conv2d/Linear 层，使用 model.named_modules() 获取完整路径名（如 "layer4.1.conv2"）。Conv2d 输出做全局平均池化压缩为 (batch, channels)。提供 extract()（detach，用于推理）和 extract_with_grad()（保留计算图，用于覆盖梯度）两个接口。

**datasets.py**：从项目 datasets/ 目录加载数据，ImageFolder 格式。seed split 取验证集的前 seed_size 张。

### 5.2 differential

**ensemble.py**：ModelEnsemble 加载多个模型，提供 predict_all()、consensus_label()、consensus_labels_batch()、has_disagreement()。target_idx 指定目标模型。

**objective.py**：DifferentialObjective 计算 obj_1(x) = Σ F_k(x)[c] - λ1·F_j(x)[c]。通过 autograd 计算对 x 的梯度。这是框架与三个开源项目的关键区别：CriticalFuzz/NeuraL-Coverage/NSGen 均使用图像变换做变异，本框架直接用 obj_1 的梯度驱动变异方向。

### 5.3 neurons

**profiler.py**：离线遍历训练集，对每个神经元计算 cl(n,T)——激活值 min-max 归一化后超过 activation_threshold 的样本占比。cl > critical_threshold 的进入关键集合 D_en。支持序列化缓存到 output/cache/，避免重复 profiling。算法参考 CriticalFuzz 但使用 PyTorch hook 而非 Keras。

**coverage.py**：CNCovTracker 跟踪哪些关键神经元已被激活。update() 返回新覆盖的神经元 ID 列表。

**objective.py**：CoverageObjective 选取未覆盖关键神经元中的第一个，以其激活值作为优化目标。当所有神经元已覆盖时返回零梯度。

### 5.4 semantic（待实现）

**protocol.py**：SemanticDistance 接口（Protocol），定义 distance() 和 batch_distance()。

```python
class SemanticDistance(Protocol):
    def distance(self, original: Tensor, mutated: Tensor) -> float: ...
    def batch_distance(self, originals: Tensor, mutateds: Tensor) -> Tensor: ...
```

**feature.py（轻量路线）**：用目标模型自身的中间层特征（如 ResNet 的 avgpool 前一层）计算原始样本与变异样本的余弦距离。不引入额外模型，一次前向传播即可。

```python
class FeatureDistance:
    def __init__(self, model: nn.Module, layer_name: str, device: torch.device): ...
```

**clip.py（重管线）**：用 CLIP 视觉编码器计算图像级语义相似度。与 NSGen 不同：NSGen 走"神经元归因 → MILANNOTATIONS 文本描述 → CLIP 文本编码器"的重管线，本方法直接用 CLIP 视觉编码器比较两张图的 embedding，省去语义解码器的依赖。

```python
class CLIPDistance:
    def __init__(self, model_name: str = "ViT-B-32", device: torch.device): ...
```

**objective.py**：SemanticObjective，计算变异样本与种子的语义距离。在 obj_total 中取负号（最小化偏移）。

```python
class SemanticObjective:
    def __init__(self, distance_fn: SemanticDistance, seed: Tensor): ...

    def __call__(self, x: Tensor) -> Tensor:
        """返回语义距离标量。"""

    def gradient(self, x: Tensor) -> Tensor:
        """返回 ∇_x obj_sem。"""
```

### 5.5 optimize

**operator.py（已实现）**：PGD 式约束——对梯度取 sign()，L∞ 裁剪（epsilon），像素范围 clamp [0,1]。

**joint.py（待实现）**：组装 obj_total = obj_1 + λ2·obj_cov - λ3·obj_sem，通过 autograd 统一计算梯度。当前 runner 中手动组合 diff 和 cov 梯度，后续应迁移到 JointObjective。

```python
class JointObjective:
    def __init__(self, diff_obj: DifferentialObjective,
                 cov_obj: CoverageObjective,
                 sem_obj: SemanticObjective,
                 lambda2: float = 0.5,
                 lambda3: float = 0.3): ...

    def __call__(self, x: Tensor, consensus_label: int) -> Tensor:
        """返回联合目标值。"""

    def gradient(self, x: Tensor, consensus_label: int) -> Tensor:
        """返回 ∇_x obj_total。"""
```

**feedback.py（待实现）**：根据 ΔCNCov、RFT、d̄_sem 三个反馈指标自适应调整 λ2 和 λ3。规则：覆盖增长饱和时降低 λ2，语义偏移过大时提升 λ3，缺陷触发率持续低时提升 λ2。

```python
@dataclass
class FeedbackState:
    delta_cncov: float
    rft: float
    d_sem_mean: float

class DynamicFeedback:
    def __init__(self, delta_cov_threshold: float = 0.01,
                 delta_sem_threshold: float = 0.5,
                 delta_fault_threshold: float = 0.05,
                 window_size: int = 5): ...

    def update(self, state: FeedbackState) -> tuple[float, float]:
        """返回调整后的 (λ_2, λ_3)。"""
```

### 5.6 engine

**seed_pool.py**：种子按 1/(fuzz_count+1) 加权采样，被选过少的种子优先。触发缺陷或覆盖新神经元的变异样本回收入池。

**runner.py**：每轮选 batch_size 个种子，每个种子做 pgd_steps 步 PGD 变异（每步计算联合梯度 → 约束 → 更新），变异后三模型推理检测分歧。支持 diff-only 和 diff+cov 两种模式。

### 5.7 evaluate

**report.py（已实现）**：save_defect_images（缺陷样本原始/变异对比图）、save_metrics（result.json）、plot_curves（缺陷发现曲线 + CNCov 曲线）。

**缺陷分类分析（待实现）**：对所有缺陷样本提取关键神经元激活向量，做聚类分析。每个簇代表一类缺陷模式，簇的数量衡量缺陷多样性。结合类别对信息（原始类别 → 错误类别），输出结构化的缺陷模式报告。与 NSGen 的区别是不依赖外部语义解码器，直接用框架自身的关键神经元体系。

**对比实验支持（待实现）**：加载多组 result.json，生成对比曲线图和汇总表格。
