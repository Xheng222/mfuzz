# 研究方案修改规划

## 写作原则

1. **叙述主语归位**：技术章节的主语是本项目，不是参考工作。参考工作以脚注式引用出现（"沿用 CriticalFuzz 的思路""参考了 NSGen 的做法"），不用整段介绍别人做了什么。
2. **研究方案 ≠ 文献综述**：每段的逻辑是"本项目做了什么设计 → 这个设计有什么参考依据"，而不是"参考工作做了什么 → 本项目在此基础上扩展"。
3. **三个核心问题**：每个技术节应回答"做什么""为什么这么做""凭什么相信能做成"，第二个是区分好坏的关键。
4. **诚实边界**：每个模块写明局限，比假装完美更有说服力。
5. **风格**：像研究者在解释自己的工作。短句为主，有主语，不堆砌形容词，不照搬审稿意见原话。

### 总体方案

- 从宏观概述改为有范围限定、技术闭环、创新点和风险边界的完整叙述
- 经历 4 轮重写确立写作风格：前两轮过于碎片化像 checklist，第三轮过度压缩充满 AI 味，第四轮找到了研究者自然叙述的语感
- 最后一轮删去了不适合出现在方案中的公式和实现细节

### 2.1 差分行为驱动

- 核心改动是补全差分测试的技术前提和判定逻辑：oracle 前提条件（同任务、同标签空间、种子预筛选）、严格缺陷判定规则（目标偏离 + 参考保持 = 候选缺陷）、PGD 式迭代过程
- 原文把分歧信号写得过于确定（"直接转化为缺陷触发信号"），修改后明确分歧只是候选信号，需要后续验证
- 修正了若干事实错误（梯度下降→上升、正确类别→共识类别）

### 2.2 中间层关键神经元识别

- 内容层面：以 CriticalFuzz 的频率定义和干预实验为基础，引入 NSGen 的归因贡献作为第二维度，构建本项目的融合关键度指标
- 补充了阈值对关键集合规模的影响、CNCov/CCCov 双指标的设计理由（全局覆盖掩盖类别不均衡）、联合梯度的耦合动机和冲突处理、缺陷分类的激活指纹方案
- 写作层面：发现参考内容占比过重（整段在讲 CriticalFuzz/NSGen 做了什么），做了叙述主语翻转——将参考工作从段落主角压缩为本项目叙述中的脚注式证据。这一原则（"主语归位"）写入了上方的写作原则，后续章节同样适用
