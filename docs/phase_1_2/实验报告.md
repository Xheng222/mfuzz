# mfuzz 框架实验报告

## 一、项目概况

本项目实现了研究方案中提出的"多维反馈驱动的中间层安全分析与测试框架"的前两个阶段。框架名为 mfuzz（multi-feedback fuzz），使用 Python 3.13 + PyTorch 2.12 (CUDA 13.2) 开发，运行环境为 Windows 11 + NVIDIA RTX 5070 Ti。

代码规模：约 1245 行（含入口脚本和测试脚本），分布在 13 个源文件中。

## 二、已实现的功能

### 2.1 差分测试模块（对应研究内容 1）

实现了基于多模型预测分歧的梯度驱动测试机制。

加载三个 ImageNet 预训练模型（ResNet50、VGG16-BN、MobileNetV2），对同一输入进行推理。当所有模型预测一致时，该预测作为共识标签。框架构建差分目标函数：

$$obj_1(\mathbf{x}) = \sum_{k \neq j} F_k(\mathbf{x})[c] - \lambda_1 \cdot F_j(\mathbf{x})[c]$$

通过 PyTorch autograd 计算该函数对输入 $\mathbf{x}$ 的梯度，以 PGD（Projected Gradient Descent）方式迭代变异输入样本，驱动目标模型偏离共识预测。

缺陷判定条件：目标模型预测偏离共识标签，且所有参考模型仍维持共识。

与现有工作的关键区别：调研的三个开源项目（CriticalFuzz、NeuraL-Coverage、NSGen）的变异策略均基于图像变换（旋转、亮度、缩放等），本框架采用梯度驱动变异。

### 2.2 关键神经元识别与覆盖引导（对应研究内容 2）

实现了关键神经元 profiling 和覆盖率跟踪机制。

离线 profiling 阶段：遍历训练集，通过 forward hook 提取目标模型（ResNet50）全部 54 个 Conv2d/Linear 层（共 27560 个神经元）的激活输出。对每个神经元计算关键度水平 $cl(n,T)$：将激活值 min-max 归一化到 [0,1]，统计超过激活阈值（0.1）的样本占比。$cl$ 超过临界阈值（0.75）的神经元进入关键集合 $\mathcal{D}_{en}$。同时按类别计算 $cl_{class}$。

在线跟踪阶段：fuzzing 过程中，每生成一个变异样本，提取目标模型中间层激活，检查是否有新的关键神经元被激活。CNCov = 已激活的关键神经元数 / $|\mathcal{D}_{en}|$。

覆盖目标 $obj_{cov}$：选取未覆盖关键神经元中激活值最低的神经元，以其输出作为优化目标，梯度引导输入向激活该神经元的方向变异。

联合优化：fuzzing 主循环使用 $obj_1 + \lambda_2 \cdot obj_{cov}$ 的联合梯度驱动变异。

### 2.3 测试引擎

实现了完整的 fuzzing 循环：

1. 从 ImageNet 验证集加载种子，筛选三个模型达成共识的样本进入种子池
2. 每轮从种子池按优先级选取种子（未充分利用的种子优先）
3. 对每个种子执行多步 PGD 变异
4. 变异后跑三模型推理，检测分歧
5. 触发缺陷或覆盖新神经元的样本回收进种子池
6. 记录 RFT、CNCov 等指标

约束算子 $\mathcal{T}(\cdot)$ 采用 PGD 式投影：L∞ 裁剪（epsilon=0.03）+ 像素范围裁剪 [0,1]。

### 2.4 输出

框架运行后输出到 `output/` 目录：

- `result.json`：指标数据（缺陷数、RFT 历史、CNCov 历史等）
- `curves.png`：缺陷发现曲线 + CNCov 增长曲线
- `defects/`：缺陷样本图片对（左原始右变异），文件名含模型预测标签

## 三、冒烟测试结果

### Phase 1 冒烟测试

使用随机张量作为输入（非真实图片），验证管线完整性。

- 模型加载：三个 ImageNet 预训练模型正常加载到 CUDA
- 激活提取：ResNet50 共 54 层，27560 个神经元
- 梯度计算：$\nabla_\mathbf{x} obj_1$ 正常返回，shape (1, 3, 224, 224)
- 约束算子：L∞ 裁剪正常
- Fuzzing 循环：5 轮迭代正常完成，耗时 1.0s
- 缺陷数：0（随机输入无共识，预期行为）

### Phase 2 冒烟测试

使用 64 张随机张量做 profiling，10 张随机种子做 fuzzing。

- Profiling：识别 27277/27560 关键神经元（99.0%）。随机数据下绝大多数神经元都"关键"是预期行为，真实数据会有更好的区分度。
- CNCovTracker：单张样本覆盖 27264 个关键神经元，CNCov=0.999
- 联合梯度 fuzzing（diff+cov）：10 轮迭代正常完成，种子池从 10 增长到 17，CNCov 跟踪正常
- 耗时：6.6s（含 profiling 约 90s）

### 真实数据初步结果（未完成）

使用 ImageNet 16 类子集（来自 NeuraL-Coverage 数据集），diff-only 模式运行了 80 轮后中断：

- 种子池：200 张中 166 张达成三模型共识
- 前 80 轮发现 365 个缺陷，种子池增长到 531
- RFT 在 0.5-0.875 之间波动，差分信号有效性初步得到验证
- 该实验未完成，无最终数据
