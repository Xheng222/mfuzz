## 实验一：纯差分 fuzzing（diff-only）

测试差分目标函数单独的缺陷发现能力，作为 baseline。

创建配置 `configs/diff_only.toml`：

```toml
[dataset]
name = "imagenet"
seed_size = 200
batch_size = 32

[models]
names = ["resnet50", "vgg16_bn", "mobilenet_v2"]
target_idx = 0

[neurons]
enabled = false

[fuzz]
max_iterations = 100
step_size = 0.01
pgd_steps = 10
batch_size = 8
lambda1 = 1.0
lambda2 = 0.0
epsilon = 0.03
log_interval = 20

[device]
name = "cuda"
```

运行：

```bash
uv run python scripts/run_fuzz.py configs/diff_only.toml
```

运行前先备份上一次的结果（output/ 会被覆盖）：

```bash
mv output output_prev  # 如果需要保留
```

预计耗时：100 轮约 3-5 分钟。

## 实验二：差分 + 覆盖引导（diff+cov）

使用默认配置，开启关键神经元 profiling 和覆盖引导。

```bash
uv run python scripts/run_fuzz.py
```

这会使用 `configs/default.toml`（neurons.enabled = true, lambda2 = 0.5）。

注意：profiling 阶段需要遍历 ImageNet 训练集，首次运行耗时较长（可能 10-30 分钟）。如果时间有限，可以减小训练集规模——在配置中暂时将 dataset.name 改为 cifar10（训练集较小）。

预计耗时：profiling + 100 轮 fuzzing 约 15-35 分钟。

## 实验三：调整迭代次数

想看更长时间的趋势，把 max_iterations 调大：

```toml
[fuzz]
max_iterations = 500

```

预计耗时线性增长。

## 查看结果

运行结束后，`output/` 目录下有：

```
output/
├── result.json     # 所有指标数据
├── curves.png      # 缺陷发现曲线 + CNCov 曲线
└── defects/        # 缺陷样本图片（左原始 右变异）
    ├── 0000_t123_r456.png
    └── ...
```

- `result.json` 中的 `defects` 字段是总缺陷数
- `rft_history` 是每轮的缺陷触发率
- `cncov_history` 是每轮的 CNCov（仅 neurons.enabled = true 时有）
- `curves.png` 直接可以截图放进组会 PPT

## 常见问题

**profiling 太慢**：ImageNet 训练集有几千张图（16 类子集），profiling 需要全部过一遍。可以先用 CIFAR-10 做快速验证。

**缺陷数为 0**：检查种子是否有共识（日志里会打印 "skipped N without consensus"）。如果大部分种子没有共识，说明三个模型在这些样本上本来就不一致，需要换种子或换模型。

**CUDA out of memory**：减小 `fuzz.batch_size`（默认 8）或 `fuzz.pgd_steps`（默认 10）。

**想换目标模型**：改 `models.target_idx`，0=ResNet50, 1=VGG16-BN, 2=MobileNetV2。
